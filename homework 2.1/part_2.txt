"Для чего используются типы данных: json, xml, yaml"
применяются для сериализации составных сложных объектов,
передачи данных клиент-сервер, создания файлов конфигурации.


XML (Extensible Markup Language)
Упрощает распределение и передачу данных.
Облегчает расширение или модернизацию операционных систем,
переход на новые приложения или браузеры без опасности потерять данные.
Используется для создания новых интернет-языков.


JSON (Java Script Object Notation)
Подходит для сериализации составных структур из списков и словарей.
Наиболее частое распространенное использование - пересылка данных от сервера к браузеру.
Обычно данные JSON доставляются с помощью AJAX, который позволяет обмениваться данными браузеру и
серверу без необходимости перезагружать страницу.


YAML (Yet Another Markup Language)
YAML в основном используется как формат для файлов конфигурации.
Применяется для настройки веб-каркасов Ruby on Rails, Dancer, Symfony,
GAE framework, Google App Engine и Dart. Также является основным языком описания классов,
ресурсов и манифестов для пакетов приложений OpenStack Murano Project и Swagger.io.
